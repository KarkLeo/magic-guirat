# Task Completion Checklist

## Что делать после завершения задачи

### 1. Проверка кода

- [ ] **Код работает** - функционал реализован согласно требованиям
- [ ] **Нет ошибок в консоли** - проверить DevTools
- [ ] **Нет warnings** от Vite или Vue
- [ ] **Код читаемый** - понятные названия, структура
- [ ] **Удалены console.log** - оставить только нужные логи

### 2. Тестирование (когда будут настроены тесты)

```bash
# Пока тестов нет, в будущем:
npm run test
npm run test:unit
```

### 3. Dev server

```bash
# Проверить что dev server работает
npm run dev

# Открыть http://localhost:5173
# Проверить в браузере:
# - Функционал работает
# - Нет ошибок в консоли
# - Hot reload работает
```

### 4. Линтинг (когда будет настроен)

```bash
# Пока нет, в будущем:
npm run lint
npm run lint:fix  # автофикс
```

### 5. Build проверка

```bash
# Проверить что production build работает
npm run build

# Проверить размер бандла
du -sh dist/

# Preview production build
npm run preview
```

### 6. Git commit

```bash
# Проверить изменения
git status
git diff

# Добавить файлы
git add <files>  # конкретные файлы
# или
git add .        # все изменения

# Commit с осмысленным сообщением
git commit -m "feat: краткое описание"
git commit -m "fix: описание фикса"
git commit -m "refactor: описание рефакторинга"

# Push (если нужно)
git push
```

### 7. Обновить .memory

```bash
# Обновить currentWork.md - статус задачи
# Если задача завершена полностью - статус: done
# Добавить заметки о решениях и паттернах
```

### 8. Специфичные проверки для Magic Guitar

#### Для аудио функционала:
- [ ] Проверить permission для микрофона
- [ ] Проверить на разных браузерах (Chrome, Firefox)
- [ ] Проверить что аудио корректно анализируется
- [ ] Проверить производительность (FPS не проседает)

#### Для визуализации:
- [ ] Проверить анимации плавные
- [ ] Проверить цветовая палитра соответствует (фиолетово-синяя)
- [ ] Проверить на разных разрешениях экрана
- [ ] Проверить производительность рендеринга

#### Performance:
- [ ] Открыть Chrome DevTools → Performance
- [ ] Записать профиль во время работы
- [ ] Проверить нет ли memory leaks
- [ ] FPS стабильный ~60fps

### 9. Документация

- [ ] Обновить README.md если нужно
- [ ] Добавить комментарии к сложному коду
- [ ] Обновить .memory файлы если архитектура изменилась

### 10. Code Review (самопроверка)

- [ ] Нет дублирования кода
- [ ] Функции небольшие и делают одну вещь
- [ ] Названия переменных понятные
- [ ] Нет захардкоженных значений (использовать константы)
- [ ] Error handling на месте
- [ ] Нет dead code

## Критерии "Done"

Задача считается завершенной когда:

✅ Функционал работает как описано в User Story  
✅ Нет ошибок и warnings  
✅ Код соответствует code style проекта  
✅ Dev server запускается без проблем  
✅ Production build успешен  
✅ Изменения закоммичены в git  
✅ currentWork.md обновлен
